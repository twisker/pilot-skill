# 隐私声明（匿名统计）

**中文** | [English](../en/privacy.md)

PILOT 内置一套**匿名使用统计**，用来回答一个问题：推荐和产品做得准不准、哪里不好用。本页面完整说明它收集什么、不收集什么、为什么收集、以及如何一键关闭。原则只有一条：**统计只描述"发生了什么类型的事"，永远不携带"你说了什么、你是谁"。**

## 当前状态：默认不上报

当前发行版的上报端点（`config/pilot.json` 的 `telemetry.endpoint`）为 **`null`**——也就是说，**目前没有任何数据离开你的机器**。事件只会写入本机队列文件 `~/.pilot/telemetry-queue.jsonl`，你可以随时打开看、随时删。

未来版本启用上报后，本页会同步更新，明确声明所使用的服务商与数据所在地；在那之前，本页其余部分描述的是启用后的完整契约。

## 收集什么

| 数据 | 说明 |
|------|------|
| 匿名安装标识 `install_id` | 首次运行时在本机随机生成的 UUID（存于 `~/.pilot/telemetry.json`），不含任何身份信息，不与账号、邮箱、设备指纹关联 |
| 事件计数（白名单六种） | `install`（安装）/ `trip_created`（创建行程）/ `export`（导出路书）/ `reco_impression`（推荐曝光）/ `reco_dismissed`（推荐被拒绝）/ `booking_link_shown`（预订链接呈现，仅带短码） |
| 目的地粗粒度 | `trip_created` 附带目的地字符串（城市/地区级，如"新疆"）与行程天数，仅此两项 |
| 导出格式 | `export` 附带格式名（pdf/xlsx/docx） |
| 产品编号与匹配度分 | `reco_impression` 附带 `product_id` 与匹配度分；`reco_dismissed` 只附带 `product_id` |

白名单在代码层强制（`tools/lib/telemetry.ts`）：白名单外的事件与字段**直接丢弃**，字符串一律截断到 200 字符——结构上排除对话内容等大块文本混入的可能。服务端（启用后）执行同一份白名单，二次过滤。

## 不收集什么

- **对话内容**：你和 PILOT 说的任何话、行程的具体内容、拒绝推荐的原因，一律不采集；
- **身份信息**：姓名、邮箱、账号、设备指纹，一概没有；
- **精确位置**：不采集 GPS/精确坐标，目的地只到城市/地区级字符串；
- **cookie 与登录态**：`~/.pilot/cookies/` 只在本机使用，绝不上传（见 [cookie 指南](guide-cookies.md)）；
- **IP 原值**：客户端统计不含 IP；启用服务端后，链接服务对访问 IP 也**只保留盐化哈希**（sha256 加盐后截取 16 位十六进制），IP 原值不落任何存储，且未配置盐值时连哈希都不记录。

## 为什么收集

只为一个目的：**改进推荐质量与产品决策**——哪些推荐被看到又被拒绝、哪种导出格式常用、安装量级如何。没有广告用途，不出售、不与第三方共享。

## 如何关闭（任一方式即全局关闭）

```bash
# 方式一：环境变量（写进 shell 配置即永久生效）
export PILOT_TELEMETRY=off

# 方式二：把 ~/.pilot/telemetry.json 的 enabled 置 false
```

关闭后所有统计调用自动变为空操作：不生成事件、不写队列、不上报。想清掉历史队列，直接删除 `~/.pilot/telemetry-queue.jsonl` 即可。

## 数据不出境说明

按当前实际情况：上报端点未配置，**所有数据留在你的本机**，不存在出境问题。未来启用上报时，将按部署所用基础设施（海外套/大陆套双端点）在本页如实声明服务商与数据存放位置，你也始终可以用上面的开关一键退出。
